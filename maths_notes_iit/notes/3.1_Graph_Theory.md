## What is a Graph? (Layman's Explanation)

A **graph** is a collection of **dots** (vertices/nodes) connected by **lines** (edges). Think of a social network: people are vertices, friendships are edges.

Formally: A graph $G = (V, E)$ where:
- $V$ = set of vertices (nodes)
- $E$ = set of edges (pairs of vertices)

---

## Types of Graphs

### Based on Edge Direction

| Type | Description | Example |
|------|-------------|---------|
| **Undirected Graph** | Edges have no direction; $(u,v) = (v,u)$ | Friendship network |
| **Directed Graph (Digraph)** | Edges have direction; $(u,v) \neq (v,u)$ | Instagram followers |
| **Mixed Graph** | Some edges directed, some not | Road network |

### Based on Other Properties

| Type | Description |
|------|-------------|
| **Simple Graph** | No self-loops, no multiple edges between same vertices |
| **Multigraph** | Multiple edges between same pair of vertices allowed |
| **Pseudograph** | Self-loops and multiple edges allowed |
| **Weighted Graph** | Each edge has a numerical weight/cost |
| **Complete Graph $K_n$** | Every pair of vertices is connected. $\binom{n}{2}$ edges |
| **Bipartite Graph** | Vertices can be split into two groups with edges only between groups |
| **Complete Bipartite $K_{m,n}$** | Every vertex in one group connected to every vertex in the other. $m \times n$ edges |
| **Null Graph** | No edges at all |
| **Regular Graph** | Every vertex has the same degree |
| **$k$-Regular Graph** | Every vertex has degree $k$ |
| **Planar Graph** | Can be drawn on a plane without edge crossings |

## Basic Terminology

| Term | Definition |
|------|-----------|
| **Degree** $\deg(v)$ | Number of edges incident to vertex $v$. Self-loop counts as 2. |
| **In-degree** $\deg^-(v)$ | (Digraph) Number of edges coming INTO $v$ |
| **Out-degree** $\deg^+(v)$ | (Digraph) Number of edges going OUT of $v$ |
| **Adjacent vertices** | Two vertices connected by an edge |
| **Incident** | An edge is incident to its endpoints |
| **Isolated vertex** | Vertex with degree 0 |
| **Pendant vertex** | Vertex with degree 1 (leaf) |
| **Walk** | Sequence of vertices with edges between consecutive vertices (repetitions allowed) |
| **Trail** | Walk with no repeated **edges** |
| **Path** | Walk with no repeated **vertices** (hence no repeated edges) |
| **Cycle** | Path that starts and ends at the same vertex |
| **Connected graph** | There is a path between every pair of vertices |
| **Component** | Maximal connected subgraph |
| **Subgraph** | Graph formed from a subset of vertices and edges |

## Handshaking Theorem (Most Important!)

$$
\sum_{v \in V} \deg(v) = 2|E|
$$

**In words:** The sum of all vertex degrees equals **twice** the number of edges.

**Corollary:** The number of vertices with **odd degree** is always **even**.

### Example
Graph with vertices A, B, C, D and edges AB, BC, CD, AD, AC.
- $\deg(A) = 3$, $\deg(B) = 2$, $\deg(C) = 3$, $\deg(D) = 2$
- Sum = $3 + 2 + 3 + 2 = 10 = 2 \times 5$ edges ✓

## Special Graphs

### Complete Graph $K_n$
- $n$ vertices, every pair connected
- Number of edges: $\binom{n}{2} = \frac{n(n-1)}{2}$
- Every vertex has degree $n-1$

| $K_1$ | $K_2$ | $K_3$ | $K_4$ | $K_5$ |
|-------|-------|-------|-------|-------|
| 0 edges | 1 edge | 3 edges | 6 edges | 10 edges |

### Complete Bipartite Graph $K_{m,n}$
- Two groups of $m$ and $n$ vertices
- Every vertex in group 1 connected to every vertex in group 2
- Edges: $m \times n$

### Cycle Graph $C_n$
- $n$ vertices forming a single cycle
- $n$ edges, every vertex has degree 2

### Petersen Graph
- Famous 3-regular graph with 10 vertices and 15 edges. Not planar.

## Graph Representations

### 1. Adjacency Matrix
For graph with $n$ vertices, an $n \times n$ matrix $A$:

$$
A[i][j] = \begin{cases} 1 & \text{if there is an edge from } v_i \text{ to } v_j \\ 0 & \text{otherwise} \end{cases}
$$

**Properties:**
- Symmetric for undirected graphs
- Row sum = degree of that vertex
- Diagonal entries = self-loops

### 2. Adjacency List
For each vertex, store a list of its neighbors.

### 3. Incidence Matrix
$n \times m$ matrix (vertices × edges). Entry is 1 if vertex is an endpoint of the edge.

## Euler Paths and Circuits

### Euler Path
A path that visits **every edge exactly once**.

**Exists if and only if:** Graph is connected and has **exactly 0 or 2 vertices** with odd degree.
- 0 odd-degree vertices → **Euler Circuit** (starts and ends at same vertex)
- 2 odd-degree vertices → **Euler Path** (starts at one odd vertex, ends at the other)

### Euler Circuit
A closed path that visits **every edge exactly once**.

**Exists if and only if:** Graph is connected and **every vertex has even degree**.

### Example
The **Königsberg Bridge Problem** — Euler proved no Euler circuit exists because all 4 vertices had odd degree.

---

## Hamiltonian Paths and Circuits

### Hamiltonian Path
A path that visits **every vertex exactly once**.

### Hamiltonian Circuit
A cycle that visits **every vertex exactly once** (and returns to start).

**Note:** There is NO simple necessary and sufficient condition (unlike Euler). This is an NP-complete problem!

### Sufficient Conditions (Dirac's & Ore's Theorems)

**Dirac's Theorem:** If $|V| \geq 3$ and every vertex has degree $\geq n/2$, then $G$ has a Hamiltonian circuit.

**Ore's Theorem:** If $|V| \geq 3$ and for every pair of non-adjacent vertices $u, v$: $\deg(u) + \deg(v) \geq n$, then $G$ has a Hamiltonian circuit.

## Trees

### Definition
A **tree** is a connected graph with **no cycles**.

### Equivalent Definitions (Any ONE implies tree)
A graph $G$ with $n$ vertices is a tree if and only if:
1. $G$ is connected and has no cycles
2. $G$ is connected and has exactly $n - 1$ edges
3. $G$ has no cycles and has $n - 1$ edges
4. There is exactly **one path** between any two vertices
5. $G$ is connected, but removing any edge disconnects it
6. $G$ has no cycles, but adding any edge creates exactly one cycle

### Properties of Trees
- A tree with $n$ vertices has exactly $n - 1$ edges
- Every tree with $\geq 2$ vertices has at least 2 **leaves** (pendant vertices)
- The sum of degrees = $2(n-1)$

## Spanning Tree

### Definition
A **spanning tree** of a graph $G$ is a subgraph that:
1. Is a **tree** (connected, no cycles)
2. Contains **all vertices** of $G$

### Properties
- A spanning tree of a graph with $n$ vertices has exactly $n - 1$ edges
- Every connected graph has at least one spanning tree
- Number of spanning trees of $K_n$ is $n^{n-2}$ (**Cayley's formula**)

### Finding Spanning Trees

#### Method 1: BFS (Breadth-First Search) Spanning Tree
1. Start from any vertex
2. Visit all neighbors first (level by level)
3. Add the tree edges used to discover new vertices

#### Method 2: DFS (Depth-First Search) Spanning Tree
1. Start from any vertex
2. Go as deep as possible before backtracking
3. Add edges that lead to unvisited vertices

---

## Minimum Spanning Tree (MST)

For a **weighted** graph, the MST is the spanning tree with **minimum total edge weight**.

### Kruskal's Algorithm
1. Sort all edges by weight (ascending)
2. Pick the smallest edge that **doesn't form a cycle**
3. Repeat until $n - 1$ edges are chosen

### Prim's Algorithm
1. Start from any vertex
2. Pick the minimum weight edge connecting a **tree vertex** to a **non-tree vertex**
3. Repeat until all vertices are in the tree

### Example

Graph with 4 vertices A, B, C, D and edges:
- AB: 1, AC: 3, AD: 4, BC: 2, BD: 5, CD: 6

**Kruskal's:** Sort: AB(1), BC(2), AC(3), AD(4), BD(5), CD(6)
- Pick AB(1) ✓
- Pick BC(2) ✓
- Pick AC(3) ✗ (forms cycle A-B-C-A)
- Pick AD(4) ✓
- Done! MST = {AB, BC, AD}, weight = 7

## Binary Trees

### Definition
A **binary tree** is a tree where every node has **at most 2 children** (left child and right child).

### Types

| Type | Description |
|------|-------------|
| **Full (Strict) Binary Tree** | Every node has 0 or 2 children |
| **Complete Binary Tree** | All levels filled except possibly the last, which is filled left to right |
| **Perfect Binary Tree** | All internal nodes have 2 children, all leaves at same level |
| **Balanced Binary Tree** | Height of left and right subtrees differ by at most 1 |

### Properties of Binary Trees

| Property | Formula |
|----------|---------|
| Max nodes at level $l$ | $2^l$ (root is level 0) |
| Max nodes in tree of height $h$ | $2^{h+1} - 1$ |
| Min height for $n$ nodes | $\lfloor \log_2 n \rfloor$ |
| For full binary tree: $n_0 = n_2 + 1$ | Leaves = Internal nodes with 2 children + 1 |
| Edges in tree with $n$ nodes | $n - 1$ |

### Tree Traversals

| Traversal | Order | Mnemonic |
|-----------|-------|----------|
| **Pre-order** | Root → Left → Right | **R**LR |
| **In-order** | Left → Root → Right | L**R**R |
| **Post-order** | Left → Right → Root | LR**R** |
| **Level-order** | Level by level, left to right | BFS |

### Example

```
        1
       / \
      2   3
     / \   \
    4   5   6
```

- **Pre-order:** 1, 2, 4, 5, 3, 6
- **In-order:** 4, 2, 5, 1, 3, 6
- **Post-order:** 4, 5, 2, 6, 3, 1
- **Level-order:** 1, 2, 3, 4, 5, 6

## Isomorphism of Graphs

Two graphs $G_1$ and $G_2$ are **isomorphic** if there is a bijection $f: V_1 \to V_2$ that preserves adjacency.

### Quick Checks (Necessary Conditions)
If any of these differ, graphs are NOT isomorphic:
1. Number of vertices
2. Number of edges
3. Degree sequence (sorted list of degrees)
4. Number of cycles of each length
5. Number of connected components

## Planar Graphs and Euler's Formula

### Euler's Formula for Connected Planar Graphs

$$
V - E + F = 2
$$

Where $V$ = vertices, $E$ = edges, $F$ = faces (regions, including the outer infinite region).

### Corollary
For a simple connected planar graph with $V \geq 3$:

$$
E \leq 3V - 6
$$

**$K_5$ is not planar:** $E = 10 > 3(5) - 6 = 9$ ✗

**$K_{3,3}$ is not planar:** $E = 9 > 3(6) - 6 = 12$... wait, $9 \leq 12$, but for bipartite: $E \leq 2V - 4 = 8 < 9$ ✗

### Kuratowski's Theorem
A graph is planar if and only if it does not contain a subdivision of $K_5$ or $K_{3,3}$.

## Graph Coloring

### Vertex Coloring
Assign colors to vertices such that no two adjacent vertices have the same color.

### Chromatic Number $\chi(G)$
The minimum number of colors needed.

| Graph | $\chi(G)$ |
|-------|-----------|
| $K_n$ | $n$ |
| $C_n$ (even) | 2 |
| $C_n$ (odd) | 3 |
| Tree | 2 |
| Bipartite | 2 |
| Petersen | 3 |
| Any planar graph | $\leq 4$ (Four Color Theorem) |

## Practice Questions

**Q1.** A graph has 8 vertices with degrees 2, 2, 3, 3, 3, 3, 4, 4. How many edges?

**Solution:** $\sum \deg = 2+2+3+3+3+3+4+4 = 24$. Edges = $24/2 = 12$.

---

**Q2.** Does the following graph have an Euler circuit? Vertices: A(deg 2), B(deg 4), C(deg 2), D(deg 4), E(deg 2).

**Solution:** All degrees are even and graph is connected → **Yes**, Euler circuit exists. ✓

---

**Q3.** Find the number of spanning trees of $K_4$.

**Solution:** By Cayley's formula: $4^{4-2} = 4^2 = 16$.

---

**Q4.** Use Euler's formula: A connected planar graph has 6 vertices and 10 edges. How many faces?

**Solution:** $V - E + F = 2 \Rightarrow 6 - 10 + F = 2 \Rightarrow F = 6$.
