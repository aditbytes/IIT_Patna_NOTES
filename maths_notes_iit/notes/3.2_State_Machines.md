## What is a Finite State Machine? (Layman's Explanation)

A **Finite State Machine (FSM)** is like a simple machine that:
1. Is always in one **state** (like a mood)
2. Reads **inputs** one at a time
3. Based on the current state and input, it **moves to a new state** and possibly produces an **output**

**Real-life example:** A traffic light. It has states (Red, Yellow, Green), inputs (timer ticks), and transitions between states based on the input.

---

## Formal Definition

A **Finite State Machine** is a 6-tuple:

$$
M = (S, I, O, f, g, s_0)
$$

Where:
| Symbol | Meaning |
|--------|---------|
| $S$ | Finite set of **states** |
| $I$ | Finite set of **input symbols** (input alphabet) |
| $O$ | Finite set of **output symbols** (output alphabet) |
| $f: S \times I \to S$ | **State transition function** (next state) |
| $g: S \times I \to O$ | **Output function** |
| $s_0 \in S$ | **Initial state** (start state) |

---

## Types of FSM

### 1. Mealy Machine
Output depends on **current state AND current input**.

$$
g: S \times I \to O
$$

Output is associated with **transitions** (edges in the diagram).

### 2. Moore Machine
Output depends on **current state only**.

$$
g: S \to O
$$

Output is associated with **states** (nodes in the diagram).

### Key Difference

| Feature | Mealy Machine | Moore Machine |
|---------|--------------|---------------|
| Output depends on | State + Input | State only |
| Output on | Transitions (edges) | States (nodes) |
| # of states needed | Usually fewer | Usually more (by 1) |
| Output timing | Changes with input | Changes with state |
| Notation | Output on arrows | Output inside states |

## Representations of FSM

### 1. State Transition Table

**Example (Mealy Machine):**

$S = \{s_0, s_1, s_2\}$, $I = \{0, 1\}$, $O = \{a, b\}$

| Current State | Input = 0 | Input = 1 |
|--------------|-----------|-----------|
| | Next State, Output | Next State, Output |
| $s_0$ | $s_1, a$ | $s_2, b$ |
| $s_1$ | $s_0, b$ | $s_2, a$ |
| $s_2$ | $s_2, a$ | $s_0, b$ |

### 2. State Transition Diagram

A directed graph where:
- **Nodes** = states (circles)
- **Edges** = transitions, labeled with input/output
- **Arrow** pointing to initial state

```
Example (Mealy):

        0/a          1/b
  (s0) ------‚Üí (s1) ------‚Üí (s2)
   ‚ÜĎ            |             |
   |   1/b      | 0/b         | 0/a (self-loop)
   |            ‚Üď             |
   ‚Üź---------- (s0)   ‚Üź------
        1/b
```

## Designing FSMs

### Example 1: Parity Checker

Design an FSM that outputs 1 when the number of 1s seen so far is **even**, and 0 otherwise.

**States:** $S = \{E, O\}$ (Even count, Odd count)

**Moore Machine:**

| State | Output | Input 0 ‚Üí Next | Input 1 ‚Üí Next |
|-------|--------|----------------|----------------|
| $E$ (start) | 1 | $E$ | $O$ |
| $O$ | 0 | $O$ | $E$ |

**Explanation:** Start at $E$ (0 ones = even). Reading a 0 doesn't change count. Reading a 1 toggles between even/odd.

### Example 2: Binary Sequence Detector

Design an FSM that outputs 1 whenever it sees the pattern "101" in the input.

**States track what we've seen so far:**
- $s_0$: Nothing useful seen (start)
- $s_1$: Seen "1"
- $s_2$: Seen "10"
- $s_3$: Seen "101" (output 1)

**Mealy Machine:**

| Current State | Input 0 | Input 1 |
|--------------|---------|---------|
| $s_0$ | $s_0, 0$ | $s_1, 0$ |
| $s_1$ | $s_2, 0$ | $s_1, 0$ |
| $s_2$ | $s_0, 0$ | $s_1, 1$ |

Note: From $s_2$ (seen "10"), if we get "1", we've seen "101" ‚Üí output 1, and go to $s_1$ (because the last "1" could start a new pattern).

**Trace for input "1101011":**

| Step | Input | Current State | Next State | Output |
|------|-------|--------------|------------|--------|
| 1 | 1 | $s_0$ | $s_1$ | 0 |
| 2 | 1 | $s_1$ | $s_1$ | 0 |
| 3 | 0 | $s_1$ | $s_2$ | 0 |
| 4 | 1 | $s_2$ | $s_1$ | **1** (found "101") |
| 5 | 0 | $s_1$ | $s_2$ | 0 |
| 6 | 1 | $s_2$ | $s_1$ | **1** (found "101") |
| 7 | 1 | $s_1$ | $s_1$ | 0 |

Output string: 0001010

**States = amount collected:** $\{s_0, s_1, s_2, s_3\}$ (‚āĻ0, ‚āĻ1, ‚āĻ2, ‚āĻ3+)

## Equivalent States

### Definition
Two states $s_i$ and $s_j$ are **equivalent** if for **every possible input string**, the machine produces the **same output** starting from either state.

$$
s_i \equiv s_j \iff \forall \text{ input strings } w: \text{output}(s_i, w) = \text{output}(s_j, w)
$$

### How to Check
States are **$k$-equivalent** if they produce the same output for all input strings of length $\leq k$.

- **0-equivalent:** Same output for the empty string (Moore) or same output for all single inputs
- **1-equivalent:** Same output for all inputs of length 1
- Refine: keep partitioning until partitions stabilize

---

## State Minimization

### Goal
Find the **minimum state machine** that behaves identically to the given machine. This is done by **merging equivalent states**.

### Algorithm (Table-Filling / Partition Refinement)

#### Step 1: Initial Partition
Group states by their output (for Moore machine):
- $P_0$: states with output 0 in one group, output 1 in another

#### Step 2: Refine Partitions
For each group, check if states transition to the **same groups** for each input:
- If two states in the same group go to **different groups** for some input ‚Üí **split** them

#### Step 3: Repeat
Continue refining until no more splits are needed.

#### Step 4: Merge
States that remain in the same group after all refinements are **equivalent**. Merge them into one state.

---

### Minimization Example

**Given Moore Machine:**

| State | Output | Input 0 ‚Üí | Input 1 ‚Üí |
|-------|--------|-----------|-----------|
| A | 0 | B | C |
| B | 0 | D | E |
| C | 1 | F | A |
| D | 0 | B | C |
| E | 1 | F | A |
| F | 1 | D | E |

**Step 1:** Initial partition by output:
- $G_1 = \{A, B, D\}$ (output 0)
- $G_2 = \{C, E, F\}$ (output 1)

**Step 2:** Refine $G_1$:
- $A$: on 0 ‚Üí $B \in G_1$, on 1 ‚Üí $C \in G_2$
- $B$: on 0 ‚Üí $D \in G_1$, on 1 ‚Üí $E \in G_2$
- $D$: on 0 ‚Üí $B \in G_1$, on 1 ‚Üí $C \in G_2$

Pattern: $A$ and $D$ both go to $(G_1, G_2)$, $B$ goes to $(G_1, G_2)$ too.

All three have same transition pattern ‚Üí No split needed.

Refine $G_2$:
- $C$: on 0 ‚Üí $F \in G_2$, on 1 ‚Üí $A \in G_1$
- $E$: on 0 ‚Üí $F \in G_2$, on 1 ‚Üí $A \in G_1$
- $F$: on 0 ‚Üí $D \in G_1$, on 1 ‚Üí $E \in G_2$

$C$ and $E$ go to $(G_2, G_1)$, but $F$ goes to $(G_1, G_2)$ ‚Üí Split!

New partition:
- $G_1 = \{A, B, D\}$
- $G_2 = \{C, E\}$
- $G_3 = \{F\}$

**Step 3:** Refine again (check $G_1$):
- $A$: 0 ‚Üí $B \in G_1$, 1 ‚Üí $C \in G_2$
- $B$: 0 ‚Üí $D \in G_1$, 1 ‚Üí $E \in G_2$
- $D$: 0 ‚Üí $B \in G_1$, 1 ‚Üí $C \in G_2$

$A$ and $D$ go to $(G_1, G_2)$, $B$ goes to $(G_1, G_2)$. Same pattern ‚Üí No split.

Check $G_2$: $C$ and $E$ both go to $(G_3, G_1)$. Same ‚Üí No split.

**Stable!** Final: $\{A, B, D\}$, $\{C, E\}$, $\{F\}$

Wait, let's re-check $G_1$ more carefully:
- $A$: 0 ‚Üí $B$, 1 ‚Üí $C$. Groups: $(G_1, G_2)$
- $B$: 0 ‚Üí $D$, 1 ‚Üí $E$. Groups: $(G_1, G_2)$  
- $D$: 0 ‚Üí $B$, 1 ‚Üí $C$. Groups: $(G_1, G_2)$

All same ‚Üí keep together.

**Minimized Machine (3 states):**
- $S_1 = \{A, B, D\}$ ‚Üí represent as $A'$
- $S_2 = \{C, E\}$ ‚Üí represent as $C'$
- $S_3 = \{F\}$ ‚Üí represent as $F'$

| State | Output | Input 0 ‚Üí | Input 1 ‚Üí |
|-------|--------|-----------|-----------|
| $A'$ | 0 | $A'$ | $C'$ |
| $C'$ | 1 | $F'$ | $A'$ |
| $F'$ | 1 | $A'$ | $C'$ |

Reduced from 6 states to 3! ‚úď

## Converting Between Mealy and Moore

### Mealy ‚Üí Moore
1. For each state-output pair in the Mealy machine, create a separate Moore state
2. The output of the Moore state is the output that was on the incoming transition

### Moore ‚Üí Mealy
1. Move the output from the destination state to the transition edge
2. The output on transition $(s_i, a) \to s_j$ becomes the output of $s_j$ in the Moore machine

## Finite Automata (DFA/NFA) ‚ÄĒ Brief Overview

### Deterministic Finite Automaton (DFA)

$$
M = (Q, \Sigma, \delta, q_0, F)
$$

| Symbol | Meaning |
|--------|---------|
| $Q$ | Finite set of states |
| $\Sigma$ | Input alphabet |
| $\delta: Q \times \Sigma \to Q$ | Transition function |
| $q_0$ | Start state |
| $F \subseteq Q$ | Set of **accepting/final** states |

**Difference from FSM:** DFA accepts or rejects strings (binary output: accept/reject) rather than producing output symbols.

| State | Output | Input 0 | Input 1 |
|-------|--------|---------|---------|
| A | 1 | B | C |

| C | 1 | B | C |

Stable ‚Üí $A \equiv C$ and $B \equiv D$. Minimized: 2 states.

## Key Formulas to Remember

| Formula | Description |
|---------|-------------|
| $M = (S, I, O, f, g, s_0)$ | FSM definition |
| Mealy: $g(S \times I) \to O$ | Output depends on state + input |
| Moore: $g(S) \to O$ | Output depends on state only |
| Partition refinement | State minimization algorithm |
| $s_i \equiv s_j$ iff same output for all inputs | Equivalence of states |
